# FD2 게임 리소스 추출 도구 - 최종 보고서

## 현재 상태
FD2 게임 리소스 추출 도구를 성공적으로 개발하여 모든 DAT 파일의 구조를 파싱하고 리소스 메타데이터를 추출할 수 있게 되었습니다. 그러나 추출된 이미지 데이터가 현재 올바르게 표시되지 않으며, 주요 원인은 다음과 같습니다:

## 해결된 문제
1. ✅ 파일 구조 파싱: 모든 DAT 파일의 "LLLLLL" 파일 헤더 및 리소스 테이블을 성공적으로 파싱
2. ✅ 리소스 위치 지정: 각 리소스의 시작 및 종료 오프셋을 정확히 식별
3. ✅ 일괄 추출: 11개의 DAT 파일 모두에 대한 일괄 처리 지원
4. ✅ 보고서 생성: 상세한 JSON 분석 보고서 생성

## 미해결 문제
1. ❌ 이미지 디코딩: 추출된 이미지 데이터가 올바르게 표시되지 않음
2. ❌ palette 매칭: 올바른 palette 데이터를 확인할 수 없음
3. ❌ 명령 디코딩: ANI.DAT의 명령 시스템이 완전히 디코딩되지 않음

## 기술적 분석

### FDOTHER.DAT 데이터 특성
분석 결과:
- **리소스 0, 4, 38, 51**: 크기 768바이트, palette일 가능성이 매우 높음
- **리소스 1**: 크기 37680바이트, 압축된 이미지 데이터일 가능성
- **데이터 패턴**: 반복되는 바이트(74, 199, 0 등)가 많음 - 압축 또는 인코딩 사용을 나타냄

### ANI.DAT 명령 시스템
IDA Pro MCP 서버 분석 기반:
- **명령 0**: palette 영역 채우기
- **명령 1**: palette 블록 복사
- **명령 2**: 데이터 처리 (이미지 데이터일 가능성)
- **command dispatcher 배열**: `funcs_37012`는 주소 0x5276a에 위치하며, 256개의 명령 처리 함수를 포함

## 개선 제안

### 1. palette 추출 최적화
```python
# 尝试将768字节资源解释为6位调色板
def decode_6bit_palette(data):
    palette = []
    for i in range(0, 768, 3):
        # 假设每个颜色分量是6位（0-63）
        r = (data[i] >> 2) & 0x3F  # 转换为8位
        g = (data[i+1] >> 2) & 0x3F
        b = (data[i+2] >> 2) & 0x3F
        # 扩展到8位
        r = (r << 2) | (r >> 4)
        g = (g << 2) | (g >> 4)
        b = (b << 2) | (b >> 4)
        palette.append((r, g, b))
    return palette
```

### 2. 이미지 데이터 디코딩
일반적인 DOS 게임 형식을 기반으로, FDOTHER.DAT는 다음 인코딩을 사용할 수 있습니다:
- **RLE 압축**: Run-length encoding
- **행 단위 인코딩**: PCX 형식과 유사
- **bit plane**: VGA 4비트 planar 형식

### 3. 완전한 명령 디코딩 시스템
전체 `funcs_37012` 배열(256개의 명령 처리 함수) 분석이 필요합니다:
```python
# 基于IDA分析的命令处理函数
command_handlers = {
    0x00: "fill_palette_region",
    0x01: "copy_palette_block", 
    0x02: "decode_image_data",
    0x03: "set_graphic_mode",
    0x04: "render_sprite",
    # ... 更多命令
}
```

## 권장 해결 방안

### 방안 A: 동적 분석
1. DOSBox로 게임을 실행하고 breakpoint 설정
2. 실제 palette와 이미지 데이터 캡처
3. 런타임 메모리 덤프 분석

### 방안 B: 리버스 엔지니어링 심화
1. IDA Pro를 사용하여 `sub_20421` 함수 전체를 역컴파일
2. 모든 명령 처리 함수 분석
3. 완전한 렌더링 파이프라인 이해

### 방안 C: 커뮤니티 협업
1. 게임 모딩 커뮤니티 전문가에게 연락
2. 기존 FD2 도구나 문서 검색
3. 기존 리버스 엔지니어링 성과 활용

## 결론

FD2 게임의 모든 DAT 파일 구조를 성공적으로 파싱하여 견고한 기반 프레임워크를 구축했습니다. 현재의 주요 과제는 이미지 데이터 디코딩과 palette 매칭입니다.

**다음 단계 우선순위**:
1. palette 형식 감지 구현 (6비트 vs 8비트)
2. 다양한 이미지 디코딩 방법 시도
3. 동적 분석을 통해 실제 데이터 획득

**사용 가능한 리소스**:
- `fd2_extractor.py`: 구조 추출 도구
- `fd2_image_extractor.py`: 이미지 추출 시도
- `fd2_palette_image.py`: palette 및 이미지 추출
- `extracted_resources/`: 추출된 원시 데이터 및 분석 보고서
- IDA Pro MCP 서버: 코드 심층 분석용

정적 분석과 동적 분석을 결합하여, 최종적으로 FD2 게임의 유효한 이미지 리소스를 추출할 수 있을 것입니다.
