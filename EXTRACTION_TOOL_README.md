# FD2 게임 리소스 추출 도구 사용 가이드

## 개요
IDA Pro MCP 서버를 통한 fd2.exe의 심층 리버스 엔지니어링 분석을 기반으로, FD2 게임 리소스 추출 도구를 개발했습니다. 이 도구는 ANI.DAT 애니메이션 리소스 파일과 FDOTHER.DAT 그래픽 리소스 파일을 파싱하고 추출할 수 있습니다.

## 파일 설명

### 주요 도구
- **fd2_extractor.py**: 주요 리소스 추출 도구
- **extracted_resources/**: 추출된 리소스 및 분석 보고서 디렉토리
- **EXTRACTION_TOOL_README.md**: 본 사용 가이드

### 생성되는 분석 보고서
1. **ani_dat_analysis.json**: ANI.DAT 파일 구조 분석 보고서
2. **fdother_dat_analysis.json**: FDOTHER.DAT 파일 구조 분석 보고서
3. **extract_all_resources.py**: 자동 생성된 완전한 리소스 추출 스크립트

## 사용 방법

### 1. 기본 추출
```bash
# 提取当前目录下的所有DAT文件
python fd2_extractor.py

# 指定工作目录
python fd2_extractor.py /path/to/game/files
```

### 2. 분석 결과 확인
추출 완료 후, `extracted_resources` 디렉토리에서:
- JSON 분석 보고서를 확인하여 파일 구조 파악
- 추출된 원시 데이터 파일 검사
- 생성된 Python 스크립트를 사용하여 일괄 추출 수행

### 3. 추가 분석
추출된 데이터를 기반으로 다음 작업이 가능합니다:
1. 추출된 이미지를 표시하는 그래픽 뷰어 개발
2. 애니메이션을 재현하기 위해 ANI.DAT 명령 시스템 파싱
3. 모든 게임 리소스(사운드, 텍스트, 레벨 데이터 등) 추출

## 기술적 세부 사항

### ANI.DAT 파일 형식
IDA Pro MCP 서버 분석 기반:
- **파일 헤더**: "LLLLLL" (6바이트)
- **리소스 테이블**: 오프셋 6에서 시작, 각 항목은 8바이트 (시작 오프셋 + 종료 오프셋)
- **리소스 구조**:
  - 8바이트 헤더 (중복 오프셋을 포함할 수 있음)
  - 173바이트 리소스 헤더
  - 블록 데이터 (각 블록은 8바이트 헤더 + 가변 길이 데이터)

### 명령 시스템
ANI.DAT는 명령 기반의 애니메이션 스크립트 시스템을 사용합니다:
- 각 블록은 하나의 명령 식별자(0-255)를 포함
- 명령 처리 함수 배열 `funcs_37012` (fd2.exe의 주소 0x5276a에 위치)
- 식별된 명령:
  - 명령 0: palette/메모리 영역 채우기
  - 명령 1: 메모리 블록 복사
  - 명령 2: 데이터 처리 (추가 분석 필요)

### FDOTHER.DAT 파일 형식
- **파일 헤더**: "LLLLLL" (6바이트)
- **리소스 테이블**: 오프셋 6에서 시작, 각 항목은 8바이트 (시작 오프셋 + 종료 오프셋)
- **리소스 데이터**: 원시 8비트 palette 인덱스 그래픽 데이터
- **일반적인 크기**: 320x200, 398x25, 32x321, 256x256 등

## 알려진 제한 사항

1. **명령 디코딩**: 일부 명령만 식별되었으며, 전체 256개 명령은 추가 분석 필요
2. **이미지 렌더링**: 추출된 그래픽 데이터는 palette 정보가 있어야 올바르게 표시됨
3. **리소스 의존성**: 일부 리소스는 다른 리소스(예: palette)에 의존할 수 있음

## 향후 개발 제안

### 단기 목표
1. 명령 디코딩 시스템 완성 (256개 명령 처리 함수 모두 분석)
2. 이미지 렌더링 기능 구현 (FDOTHER.DAT의 palette와 결합)
3. BG.DAT, FIGANI.DAT 등 다른 DAT 파일 지원 추가

### 장기 목표
1. 완전한 리소스 브라우저 개발
2. 애니메이션 플레이어 구현
3. 리소스 재패키징 지원

## 문제 해결

### 일반적인 문제
1. **파일을 찾을 수 없음**: ANI.DAT와 FDOTHER.DAT가 작업 디렉토리에 있는지 확인
2. **메모리 부족**: 대용량 파일 처리 시 Python 메모리 제한을 늘려야 할 수 있음
3. **인코딩 문제**: 보고서의 16진수 데이터는 추가 파싱이 필요할 수 있음

### 디버깅 팁
- 생성된 JSON 보고서를 확인하여 추출 과정 파악
- `extracted_resources` 디렉토리의 파일 검사
- 특정 함수를 추가 분석하려면 IDA Pro MCP 서버 사용

## 연락 및 지원
IDA Pro MCP 서버의 분석 결과를 기반으로, 이 도구는 오픈 소스 프로젝트입니다. 문제나 개선 제안이 있으면 프로젝트 문서를 참조하세요.
