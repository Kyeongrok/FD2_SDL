# FD2.EXE 역공학 함수 기능 설명 문서

본 문서는 IDA Pro 디스어셈블 분석과 decompile 도구로 추출한 의사 코드를 기반으로, FD2.EXE 안의 역공학 함수들을 기능별로 분류하고 설명합니다.

---

## 1. 데이터 파일 로드 모듈

### sub_111BA (0x111BA)
**기능**: 범용 DAT 데이터 파일 로드 함수

**설명**: 지정된 DAT 파일에서 데이터 레코드를 읽는다. 파일을 먼저 열고, 레코드 인덱스로 오프셋을 계산해 레코드 포인터와 크기를 읽은 뒤, 메모리를 할당하고 전체 데이터를 읽는다.

**호출 관계**:
- 호출됨: FDTXT.DAT, FDFIELD.DAT, FDSHAP.DAT 등 데이터 파일 읽기
- 호출함: fopen, malloc, fseek, fread

---

## 2. 게임 유닛 / 스프라이트 로드 모듈

### sub_1088D (0x1088D)
**기능**: 게임 스테이지 / 유닛 데이터 초기화 로드

**인자**: `n13` - 스테이지 / 씬 인덱스

**설명**: 지정 스테이지의 모든 게임 리소스를 로드한다. 포함 내용:
- FDTXT.DAT 에서 텍스트 / 대화 데이터 로드
- FDFIELD.DAT 에서 맵 / 전장 데이터 로드 (서로 다른 3개 인덱스)
- FDSHAP.DAT 에서 스프라이트 스프라이트 데이터 로드
- FDICON.B24 아이콘 파일 읽기
- 7680바이트 유닛 버퍼 할당
- 유닛 배열 초기화, 유닛당 80바이트

**핵심 데이터 구조**:
- `dword_53A51` - 맵 데이터 포인터
- `dword_53A55` - 스프라이트 인덱스 데이터
- `dword_53A59` - 스프라이트 데이터 포인터
- `dword_53A45` - 유닛 배열 버퍼 (7680 = 96유닛 × 80바이트)

---

### sub_10B4E (0x10B4E)
**기능**: 게임 유닛 생성 및 초기화

**설명**: 현재 스테이지 데이터를 기반으로 유닛 배열을 생성하며, 모든 유닛을 순회하면서 조건에 맞는 유닛에 대해 sub_10C50 을 호출해 스프라이트 데이터를 로드한다.

**호출 흐름**:
- FDFIELD.DAT 을 열어 유닛 정의 획득
- 모든 유닛 (dword_53BE3 개) 순회
- 타입이 일치하는 유닛 필터링 (a5 인자)
- 각 유닛에 대해 sub_10C50 호출하여 스프라이트 로드
- FD2.TMP 파일로 출력

---

### sub_10C50 (0x10C50)
**기능**: 단일 유닛의 스프라이트 프레임 데이터 로드

**인자**: `a5` - 유닛 인덱스, `a6` - 파일 포인터

**설명**: 단일 게임 유닛에 대해 스프라이트 프레임 데이터를 로드한다. 포함 내용:
- 유닛의 버퍼 내 위치 계산
- 맨해튼 거리 계산으로 유닛 방향 결정
- FDSHAP.DAT 에서 스프라이트 스프라이트 데이터 로드
- 유닛의 위치, 방향, 애니메이션 프레임 등 정보 설정
- 80바이트 유닛 구조체 채우기

**유닛 구조 (80바이트)**:
```
+0: x 좌표
+1: y 좌표
+2: 스프라이트 인덱스
+3: 방향 플래그
+4: 애니메이션 프레임
+5: 타입 플래그
+6: 유닛 타입
+7: 스프라이트 인덱스
+8: 스프라이트 인덱스
+10-12: 색상 / 효과 데이터
+14-24: 애니메이션 데이터
+31-33: 위치 증분값
+34-39: 초기화
+49: 플래그
+50-53: 애니메이션 파라미터
+55-70: 크기 데이터
```

---

### sub_112A5 (0x112A5)
**기능**: 지정 위치에 유닛 레코드 생성

**인자**: `n0x44` - 유닛 타입 인덱스

**설명**: 전장 배열에 새로운 유닛 레코드를 생성하고 다음을 설정:
- 유닛 타입 (n0x44)
- 위치 정보
- 기본 애니메이션 파라미터
- 히트 포인트 / 충돌 박스 데이터

---

### sub_1145A (0x1145A)
**기능**: 유닛 충돌 박스 / 바운딩 박스 갱신

**인자**: `a5` - 유닛 인덱스

**설명**: 유닛의 현재 위치와 부속 스프라이트들을 기반으로 유닛의 충돌 바운딩 박스를 계산하고 갱신한다. 유닛의 모든 부속 스프라이트(최대 8개)를 순회하며 그들의 경계를 누적한다.

## 3. 입력 처리 모듈

### sub_11AA8 (0x11AA8)
**기능**: 키보드 입력 폴링

**설명**: 키보드 입력 상태를 대기하고 읽는다. DOS 인터럽트 22h 를 통해 키보드 스캔 코드를 획득하고 다음을 반환:
- 28 (0x1C) - Enter 키
- 1 (0x01) - ESC 키
- 82 (0x52) - 숫자패드 Ins
- 224 (0xE0) - 확장 스캔 코드 접두사

---

### sub_115B6 (0x115B6)
**기능**: 메인 게임 입력 처리 함수

**인자**: `n6` - 입력 모드, `n6_3` - 유닛 인덱스, `a7` - 입력 버퍼

**설명**: 다양한 게임 입력 처리. 포함 내용:
- 방향키 이동 (n6 == 5/6)
- 액션 키 (K=75, M=77, P=80, H=72)
- 공격 명령 처리
- 유닛 선택 및 이동

**입력 코드**:
- 57/28: 확인 / 선택
- 44/76: 왼쪽 이동
- 72: 위 이동 / 공격
- 80: 오른쪽 이동
- 75: 왼쪽 이동
- 77: 메뉴 / 취소

---

### sub_117E7 (0x117E7)
**기능**: 메인 게임 루프 / 이벤트 디스패치

**설명**: 게임 메인 루프, 입력과 게임 로직 처리:
- 키보드 입력 감지 (44/76/57/28)
- 유닛 검색 및 선택
- 공격 애니메이션 재생
- 특수 효과 처리
- 함수 포인터 테이블 호출 (funcs_1197B, funcs_1199C)

---

## 4. 유닛 이동 모듈

### sub_11B48 (0x11B48)
**기능**: 유닛을 아래로 이동

**설명**: 선택된 유닛을 한 칸 아래로 이동하며 동시에 y 좌표와 애니메이션 인덱스를 감소시킨다.

---

### sub_11B9B (0x11B9B)
**기능**: 유닛을 오른쪽으로 이동

**설명**: 선택된 유닛을 오른쪽으로 이동하며 y 좌표와 애니메이션 인덱스를 증가시킨다.

---

### sub_11C59 (0x11C59)
**기능**: 유닛을 왼쪽으로 이동

**설명**: 선택된 유닛을 왼쪽으로 이동하며 x 좌표와 애니메이션 인덱스를 감소시킨다.

---

## 5. 렌더링 / 그리기 모듈

### sub_11CAC (0x11CAC)
**기능**: 메인 렌더링 함수 - 게임 화면 그리기

**설명**: 핵심 렌더링 함수로 다음을 담당:
- sub_1297D 호출하여 렌더링 데이터 준비
- sub_11EEE 호출하여 맵 / 배경 렌더링
- sub_122DC 호출하여 전투 / 특수 효과 처리
- sub_127A9 호출하여 유닛 렌더링
- sub_1ACF3 호출하여 전경 렌더링
- sub_11EB0 호출하여 실제 메모리 복사 / 디스플레이 실행

---

### sub_11EB0 (0x11EB0)
**기능**: 메모리 블록 이동 / 렌더링 데이터 전송

**설명**: 소스 주소에서 목적지 주소로 데이터 복사. 용도:
- 화면 버퍼 갱신
- 스프라이트 / 배경 데이터 복사
- 비디오 메모리 연산

---

### sub_122DC (0x122DC)
**기능**: 전투 특수 효과 렌더링

**인자**: `dword_51A83` - 특수 효과 타입

**설명**: 특수 효과 타입에 따라 폭발 / 공격 효과를 그림:
- case 1: 단일 지점 공격
- case 2: 수평 공격
- case 3: 십자 공격 (5 방향)
- case 4: 확장 십자 공격 (13 방향)
- case 5: 초광범위 공격 (18 방향)
- case 6: 맵 마커 클리어

---

### sub_127E0 (0x127E0)
**기능**: 사운드 이펙트 트리거

**설명**: 유닛 위치와 타입에 따라 대응 사운드 이펙트를 트리거:
- 키보드 포트 0x46C 를 읽어 상태 변화 감지
- 유닛 타입 (v5[3], v5[4]) 에 따라 사운드 소스 오프셋 계산
- sub_4E1A6 또는 sub_4E22A 를 호출해 사운드 재생

---

### sub_1300D (0x1300D)
**기능**: 공격 애니메이션 시퀀스 재생

**인자**: `a5` - 유닛 인덱스

**설명**: 유닛의 공격 애니메이션을 재생:
- 6회 루프 (n7: 1-6)
- 각 루프: sub_32230, sub_1297D, sub_11EEE, sub_127A9, sub_11EB0 호출
- 유닛 위치와 애니메이션 프레임 갱신

---

### sub_135DD (0x135DD)
**기능**: 이동 애니메이션 처리

**설명**: 유닛 이동 시의 애니메이션 처리:
- 유닛의 x 좌표를 목표값이 될 때까지 루프 조정
- 유닛의 y 좌표를 목표값이 될 때까지 루프 조정
- 이동마다 sub_11CAC 호출하여 렌더링

---

### sub_1366A (0x1366A)
**기능**: 복잡한 애니메이션 / 공격 시퀀스 처리

**인자**: `a5` - 애니메이션 데이터 인덱스

**설명**: 복잡한 애니메이션 시퀀스 처리:
- 애니메이션 커맨드 데이터 파싱
- 두 가지 애니메이션 모드 지원:
  1. 루프 애니메이션: 애니메이션 프레임 반복 재생
  2. 위치 애니메이션: 데이터에서 위치 정보 로드
- 공격 애니메이션과 특수 효과 지원

---

### sub_145CD (0x145CD)
**기능**: 스프라이트 가시성 체크

**설명**: 모든 유닛을 순회하며 스프라이트 가시성 체크:
- 유닛이 가려졌는지 체크
- 유닛이 화면 범위 안에 있는지 체크
- sub_14625 호출하여 비가시 유닛 처리

---

## 6. 전투 / 충돌 모듈

### sub_14EF0 (0x14EF0)
**기능**: 전투 / 충돌 판정

**설명**: 유닛 간 거리와 상대 위치에 따라 전투 결과 판정:
- 대상 유닛 거리 계산
- 서로 다른 방향의 우선순위 비교
- 대응 처리 함수 호출:
  - sub_15311: 원거리 공격
  - sub_1548E: 특수 공격
  - sub_15055: 돌격 공격

---

## 7. 사운드 / 음악 모듈

### sub_11D40 (0x11D40)
**기능**: OPL 음악 칩 프로그래밍

**설명**: OPL 오디오 칩 (포트 968, 969) 에 음악 데이터 쓰기:
- 0-255 의 음악 시퀀스 루프 출력
- FDOTHER.DAT 에서 음악 데이터 로드
- 각 음표에 볼륨 조정 적용

---

### sub_1B750 (0x1B750)
**기능**: 유닛 충돌 박스 / 바운딩 박스 계산

**인자**: `a5` - 유닛 인덱스

**설명**: 유닛의 충돌 바운딩 박스 데이터를 계산하고 갱신:
- 유닛 구조체 오프셋 55, 57, 62 에서 기본 경계값 읽기
- 오프셋 36 의 플래그 비트 체크, 있으면 v11 에 15 를 더함
- 가능한 8개 부속 스프라이트를 모두 순회 (오프셋 10-11, 플래그 0x40 루프 체크)
- 활성화된 각 부속 스프라이트에 대해 그 경계 데이터 누적
- 오프셋 34/35 에 플래그 비트가 있으면, 경계값에 부동소수점 스케일링 적용 (_CHP 명령)
- 최종 계산 결과를 유닛 구조체 오프셋 72, 74, 76 에 되쓰기

---

### sub_11019 (0x11019)
**기능**: 스프라이트 프레임 데이터 캐시 관리

**인자**: `a1` - 프레임 인덱스, `a2` - 파일 포인터

**설명**: FDICON.B24 에서 스프라이트 프레임 데이터를 로드하고 캐시 관리:
- 6720바이트 임시 버퍼 할당
- 프레임 인덱스 테이블 읽기 (13개 DWORD)
- 프레임이 캐시되어 있는지 체크 (dword_53BDF > 0)
- 캐시되어 있으면 캐시 인덱스 반환
- 캐시되어 있지 않으면 메인 캐시 영역 할당 (dword_53A61)
- 프레임 데이터를 캐시에 로드하고, 12개 서브프레임의 오프셋 포인터 계산
- 최대 dword_53BDF 개의 캐시 프레임 지원

---
