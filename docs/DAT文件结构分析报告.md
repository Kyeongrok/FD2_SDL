# 炎龙骑士团II DAT文件结构分析报告

## 概述

本文档基于IDA Pro逆向工程对FD2.EXE的深度分析，验证并补充了各个DAT文件的数据结构。所有DAT文件采用统一的索引表+数据块存储模式，但具体索引结构有所不同。

---

## 通用文件格式

### 通用头部结构（所有DAT文件）

```
+------------------+ <- 0
| 魔数 "LLLLLL"   |  6字节
+------------------+ <- 6
| 索引条目数(X)    |  4字节 (little-endian uint32)
+------------------+ <- 10
| 资源偏移表       |  每个条目8字节: [起始偏移, 结束偏移]
| [起始, 结束]     |
| [起始, 结束]     |
| ...              |
+------------------+ <- 10 + X*8
| 资源数据区       |
+------------------+
```

### 核心加载函数分析

通过分析`sub_111BA`函数（处理DAT文件读取的核心函数），验证了文件结构：

```c
// IDA反编译代码 (地址 0x111BA)
void __fastcall sub_111BA(..., int a7, ...) {
    fseek(_rb_, 4 * a7 + 6, 0);   // 定位到索引位置: 4*索引号 + 6
    sub_373CA(v8, 1u, 8, _rb_);   // 读取8字节
    v9 = *v8;                      // 起始偏移 (startOffset)
    dword_53BFF = v8[1] - *v8;   // 大小 = 结束偏移 - 起始偏移
    
    // 分配内存并读取数据
    fseek(_rb_, v9, 0);            // 跳转到数据起始位置
    sub_373CA(v10, 1u, dword_53BFF, _rb_); // 读取数据块
}
```

**关键验证点**：
- 索引位置计算公式: `offset = 4 * resourceIndex + 6`
- 每个索引条目: 8字节 = 4字节起始偏移 + 4字节结束偏移
- 数据块大小: `size = endOffset - startOffset`

## RLE压缩格式（已验证）

基于逆向代码验证的RLE格式：

| 控制码高2位 | 模式 | 含义 |
|------------|------|------|
| 00 | 填充 | 用下一字节填充 (num & 0x3F) + 1 次 |
| 01 | 字面 | 直接复制 (num & 0x3F) + 1 字节 |
| 10 | 填充 | 与00相同 |
| 11 | 跳过 | 仅移动指针 (num & 0x3F) + 1 字节 |

**解压算法**（C代码）：

```c
void decompress_rle(byte* src, int src_size, byte* dst, int stride) {
    word count = *(word*)src;      // 每行字节数
    word num_rows = *(word*)(src + 2);  // 行数
    
    src += 4;
    int pos = 0;
    
    for (int row = 0; row < num_rows; row++) {
        int remaining = count;
        
        while (remaining > 0 && pos < src_size) {
            byte code = src[pos++];
            int num = (code & 0x3F) + 1;
            if (num > remaining) num = remaining;
            
            int type = (code >> 6) & 0x3;
            
            switch(type) {
                case 0: // 00: fill
                case 2: // 10: fill
                    byte fill = src[pos++];
                    memset(dst, fill, num);
                    dst += num;
                    break;
                case 1: // 01: literal
                    memcpy(dst, src + pos, num);
                    pos += num;
                    dst += num;
                    break;
                case 3: // 11: skip
                    dst += num;
                    break;
            }
            remaining -= num;
        }
        dst += stride - count;
    }
}
```

## 验证方法

通过IDA Pro MCP分析FD2.EXE的关键函数验证：

1. **sub_111BA** (0x111BA): 通用DAT文件加载函数
   - 确认索引位置公式: `4 * index + 6`
   - 确认每条目8字节格式: `[start, end]`

2. **字符串引用** (0x51A40-0x51A79):
   - 确认DAT文件名列表

3. **main函数** (0x25BF4): 调用各加载函数

## 结论

通过IDA逆向工程验证，确认了：
1. 所有DAT文件使用统一的"LLLLLL"魔数和索引表结构
2. 索引位置公式: `offset = 4 * resourceIndex + 6`
3. 每索引条目8字节: [起始偏移, 结束偏移]
4. 偏移6处的值(X)表示索引表条目数
5. 实际资源数量可能少于索引条目数

大部分文件已可成功解析出图像资源，剩余文件需要进一步分析其特有的子索引结构。
