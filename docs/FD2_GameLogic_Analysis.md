**핵심 발견** (`sub_12C0D` 출처):
- `offset+0`: 팀 플래그 (dword_53AB1 과 비교)
- `offset+1`: 미상
- `offset+5`: 상태 플래그 (0x85 와 비교)
- `offset+6`: AI 타입 (2 = 적)
- `offset+7`: 아이콘 ID

## 2. 게임 메인 루프 구조

### 2.1 main 함수 (0x25E3A)

```c
while (1) {
    sub_25977(18, 0);      // 게임 상태 초기화 / 음악 재생
    v10 = sub_25EBB();     // 세이브 체크 / 게임 진입
    
    if (v10 == 0) {
        // 게임 루프 계속
        do {
            v10 = sub_117E7();  // 입력 처리 / 상태 머신
            
            if (n2_0 == 1) {
                // 새 상태로 진입
                sub_22E5C();
            }
            else if (n2_0 == 2) {
                // 상태 처리 함수 실행
                funcs_25E23[n17]();  // 상태 처리 함수 테이블
            }
        } while (!v10);
    }
    
    if (v10) {
        // 게임 종료
        sub_37ED8();
    }
}
```

## 4. 게임 상태 머신

게임은 `n17` 변수에 현재 게임 상태를 저장하며, `funcs_25E23` 과 `funcs_25E3A` 는 함수 테이블입니다.

```
상태 0:  타이틀 / 시작
상태 1:  오프닝 애니메이션
상태 2:  메뉴
상태 3:  스테이지 선택
상태 4:  전투 / 게임 중
상태 5:  일시정지
상태 6:  승리 화면
```

## 6. DAT 파일 로드

`sub_111BA` 는 범용 DAT 파일 로더 함수:

```c
void* sub_111BA(const char* filename, void* prev_ptr, int resource_index) {
    // offset = 4 * index + 6
    fseek(file, 4 * resource_index + 6, SEEK_SET);
    // 8바이트 읽기: [start_offset, end_offset]
    size = end_offset - start_offset;
    // 메모리 할당 및 데이터 읽기
    return data;
}
```
