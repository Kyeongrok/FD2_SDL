# FD2.EXE 시작 흐름 완전 분석

## 1. 진입점과 프로그램 아키텍처

### 진입점
- **주소**: 0x3CCB4 (start)
- **종류**: 라이브러리 함수, 디컴파일되지 않음
- **호출**: DOS/4G 런타임 환경에서 호출되어 main() 으로 점프

### main() 함수
- **주소**: 0x25BF4
- **호출자**: 0x4609B (C 런타임 시작 코드)

  ├─ sub_3908B()                      # 오디오 드라이버 초기화
  │    └─ 반환: dword_53EDC (드라이버 핸들)
  │
  ├─ sub_392D0(dword_53EDC)          # 오디오 핸들 할당

## 3. 메인 게임 루프와 애니메이션 재생

### 3.1 메인 루프 구조
```

  └─ while(1)                         # 메인 루프

       ├─ sub_25EBB()                 # 세이브 체크 / 게임 상태 진입
       │    ├─ sub_1F894()           # 오프닝 애니메이션 재생 및 메뉴 표시
       │    ├─ sub_10010()           # 세이브 데이터 로드
       │    └─ sub_25977()           # 재초기화
       └─ sub_117E7()                # 상태 머신 / 메뉴 처리
```

### 3.2 오프닝 애니메이션 재생 (sub_1F894 - 0x1F894)

#### 로드되는 리소스:
```
sub_1F894()
  ├─ FDOTHER.DAT (리소스 77)          # 애니메이션 데이터 로드
  ├─ FDOTHER.DAT (리소스 76)          # 배경 이미지 로드
  ├─ FDOTHER.DAT (리소스 74)          # 타이틀 이미지 로드
  ├─ FDOTHER.DAT (리소스 76)          # 배경 재로드
  ├─ FDOTHER.DAT (리소스 101)         # 메뉴 배경 로드
  └─ FDOTHER.DAT (리소스 69-73)       # 5프레임 애니메이션 로드
```

#### 애니메이션 재생 흐름:
1. **화면 초기화**
   - `memset(655360, 0, 64000)` - 비디오 메모리 클리어
   - `sub_11D40()` - 팔레트 설정

2. **타이틀 표시**
   - `sub_4E98D()` - BG 이미지를 비디오 버퍼로 압축 해제
   - `sub_1F525()` - 화면에 복사
   - `sub_17AA9()` - 비디오 효과 처리

3. **스크롤 애니메이션 재생**
   - 애니메이션 프레임용 230,400바이트 버퍼 할당
   - 5프레임 애니메이션 로드 (FDOTHER.DAT 리소스 69-73)
   - 메인 루프 (535회 반복):
     ```
     for (n535 = 535; n535 >= 0; n535--)
     {
       sub_11EB0()           # 프레임을 비디오 버퍼로 복사
       sub_1F525()           # 화면에 표시
       
       # 특정 프레임에서 효과 삽입
       if (n535 == 450) sub_1F73F()  # 특수 효과
       if (n535 == 330) sub_1F81E()  # 사운드 이펙트 트리거
       if (n535 == 210) sub_1F81E()
       if (n535 == 110) sub_1F81E()
       if (n535 == 10)  sub_1F73F()
       
       sub_10620()           # 키 체크 (애니메이션 스킵 가능)
       j___delay(30)         # 프레임 지연
     }
     ```

4. **페이드 아웃 효과**
   - `sub_2DF01()` - 팔레트 페이드 아웃 (40단계)
   - 애니메이션 버퍼 해제

5. **메뉴 표시**
   - FDOTHER.DAT (리소스 7, 8) 로드 - 메뉴 데이터
   - `sub_20421(1, 15, 1)` - 메뉴 텍스트 표시
   - `sub_25B45()` - 메뉴 옵션 표시
   - `sub_11DF2()` - 팔레트 설정

6. **메뉴 루프**
   ```
   while (!n12)
   {
     sub_1FF79()              # 메뉴 옵션 그리기
     int386(22, &n3, &n3)     # 키보드 입력 획득
     
     if (key == UP) n3--;     # 옵션 위로
     if (key == DOWN) n3++;   # 옵션 아래로
     if (key == ENTER/SPACE) n12 = 1;  # 선택 확인
   }
   ```

7. **메뉴 선택 확인**
   - 깜빡임 효과 (4회)
   - `sub_1F882()` - 정리
   - `memset(655360, 0, 64000)` - 화면 클리어
   - 선택 결과 반환

## 8. 비고

1. **비디오 메모리 레이아웃**: 0xA0000 (655360) 은 VGA 모드 13h 의 비디오 버퍼 시작 주소
2. **DAT 파일 포맷**: 모든 DAT 파일은 통일된 인덱스 포맷 사용: 6바이트 헤더 + 리소스당 4바이트 인덱스 테이블
3. **팔레트**: 6비트 팔레트 값을 사용하며, 8비트로 변환하려면 2비트 좌 시프트 필요
4. **메모리 관리**: DOS 모드에서 세그먼트 주소 지정 사용, 모든 포인터는 32비트 선형 주소
