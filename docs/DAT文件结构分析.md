# 용의기사2 DAT 파일 데이터 구조 분석

## 개요

본 문서는 용의기사2 리소스 파일 중 각 DAT 파일의 헤더 데이터 구조와 로드/파싱 흐름을 기술한다. 모든 파일은 유사한 인덱스 테이블 + 데이터 블록 저장 방식을 사용하지만, 구체적인 인덱스 구조는 파일마다 다르다.

---

## 공통 데이터 구조

### DataBlock 구조
```python
class DataBlock:
    startOffset: int  # 데이터 블록 시작 오프셋
    length: int       # 데이터 블록 길이
```

### 공통 파일 헤더 포맷
대부분의 DAT 파일은 6바이트 헤더로 시작한다:
- **바이트 0-1**: 미상 / 예약
- **바이트 2-5**: 첫 번째 데이터 블록의 시작 오프셋 (보통 6, 즉 헤더 다음)

## 1. FDICON.B24 - 인물 아이콘

### 파일 구조
```
+------------------+ <- 0
| 파일 헤더 (6바이트) |
+------------------+ <- 6
| 인덱스 테이블 (1681항목) |  항목당 4바이트 uint32, 총 6724바이트
| [오프셋 0]        |
| [오프셋 1]        |
| ...              |
| [오프셋 1680]     |
+------------------+ <- 6730
| 데이터 영역       |
| [아이콘 0 데이터]  |
| [아이콘 1 데이터]  |
| ...              |
+------------------+
```

### 인덱스 구조
- **인덱스 개수**: 1681 아이콘
- **인덱스 위치**: 오프셋 6부터, 4바이트마다 uint32
- **데이터 블록 계산**: `DataBlock(array[i], array[i+1] - array[i])`
- **마지막 블록**: `DataBlock(array[1680], 파일 길이 - array[1680])`

### 파싱 흐름
```
load_fdicon_file() 
  -> fileDatas 로 파일 읽기
  -> AnalysisICON()
      -> 인덱스 테이블 루프 읽기 (num = 6; num <= 6726; num += 4)
      -> 1681개 DataBlock 생성
  -> generate_icon_images()
      -> makeShapBMP(24, 24, ...) 로 24x24 아이콘 생성
```

## 2. FDTXT.DAT - 텍스트 리소스

### 파일 구조 (2단계 인덱스)
```
+------------------+ <- 0
| 파일 헤더 (6바이트) |
+------------------+ <- 6
| 메인 인덱스 테이블 (35항목) |  항목당 4바이트 uint32, 총 140바이트
| [오프셋 0]        |
| [오프셋 1]        |
| ...              |
| [오프셋 34]       |
+------------------+
| 서브 블록 개수 영역 |  각 메인 카테고리 시작에 서브 블록 개수
| 서브 블록 오프셋 테이블 | 각 메인 카테고리에 4바이트 오프셋 여러 개
+------------------+
```

### 인덱스 구조
- **메인 카테고리 개수**: 35
- **메인 인덱스 테이블**: 오프셋 6부터, 4바이트마다 uint32
- **서브 블록 개수**: 각 메인 카테고리 시작 위치에서 short 값 읽기 `/ 2`
- **서브 블록 오프셋 테이블**: 각 메인 카테고리에 N개의 서브 블록 오프셋

### 파싱 흐름
```
load_fdtxt_file()
  -> fileDatas 로 파일 읽기
  -> AnalysisTXT()
      -> 35개 메인 오프셋을 array 로 읽기
      -> 각 메인 카테고리에 대해:
          -> 서브 블록 개수 읽기 (array[i] 위치의 short / 2)
          -> 서브 블록 오프셋 배열 읽기
          -> DataBlock 배열 생성
  -> generate_fdtxt_files()
      -> 각 서브 블록에 대해 makeWord() 호출하여 텍스트 파싱
```

## 이미지 데이터 포맷

### makeFaceBMP/makeBgBMP/makeTAIBMP/makeFightBMP
인물 표정, 전투 배경, 전투 동작 등에 사용:
- **오프셋 0-1**: 너비 (short)
- **오프셋 2-3**: 높이 (short)
- **오프셋 4+**: 압축 이미지 데이터

### makeShapBMP
스프라이트 리소스와 아이콘에 사용:
- 너비/높이가 사전 정의되지 않음 (고정 크기)
- 데이터 포맷은 makeFaceBMP 와 동일

### makeANIBMP
애니메이션 프레임에 사용:
- **오프셋 0-1**: 프레임 데이터 길이 (ushort)
- **오프셋 2-3**: 너비 (ushort)
- **오프셋 4-5**: 높이 (ushort)
- **오프셋 6+**: 압축 이미지 데이터

---

## RLE 압축 포맷

이미지 데이터는 개선된 RLE 압축을 사용한다:

| 바이트 값 범위 | 의미 |
|-----------|------|
| 0-63 | 단일 픽셀 반복, (값+1) 번 그리기 |
| 64-127 | 연속 그리기, (값-64) 번 그린 후 x 좌표 +1 |
| 128-191 | 픽셀 반복, 다음 바이트를 (값-128+1) 번 반복 |
| 192-255 | 픽셀 스킵, (값-192+1) 개 스킵 |
